# Fairshot

> A Soft Tooling product. Job search without the shame spiral.
> _We run your job search. You pay when it works._

A [TanStack Start](https://tanstack.com/start) app porting the Fairshot prototypes
(marketing landing + dashboard) into a single full-stack React application with
SSR and file-based routing. The original standalone HTML/React prototypes and the
product/brand/design docs are preserved under [`design-source/`](./design-source).

## Stack

- **TanStack Start** (Vite 8 + Nitro) — SSR + file-based routing
- **TanStack Router** — type-safe routing
- **React 18**
- **[Panda CSS](https://panda-css.com/) + [Park UI](https://park-ui.com/) + [Ark UI](https://ark-ui.com/)** —
  type-safe, co-located styles with the Park UI design system as the token source
  (teal accent, sand neutrals, `xl` radius). Recipes live in `styled-system/` and
  are wrapped as React primitives in [`src/components/ui/`](./src/components/ui).
- Editorial typography — _Newsreader_ display + _Inter_ body + _SF Mono_ eyebrow,
  warm cream palette with a soft teal glow on the app canvas.
- **Feature-based architecture** — enforced by `npm run lint:arch`.

## Routes

| Path             | Screen                                                          |
| ---------------- | -------------------------------------------------------------- |
| `/`              | Marketing landing — hero, problem, how-it-works, pricing, waitlist |
| `/app`           | Dashboard shell (sidebar + milestone bar) → redirects to Resume |
| `/app/resume`    | Resume reframer (paste background → stronger framing)          |
| `/app/discover`  | Role discovery grid, scored on transparency / response / fit   |
| `/app/tracker`   | Application Kanban: Applied → Screening → Interview → Offer     |

Applying to a role on **Discover** adds it to the **Tracker** and advances the
milestone bar. Shared state lives in [`src/lib/applications.tsx`](./src/lib/applications.tsx)
(React context), mounted by the `dashboard` feature so the sidebar count, milestone
bar, and screens stay in sync.

## Develop

```bash
npm install
npm run dev        # http://localhost:3000
npm run typecheck  # tsc --noEmit
npm run lint:arch  # validate feature-based architecture
```

## Production

```bash
npm run build    # client + SSR bundle into dist/ and .output/
npm run start    # node .output/server/index.mjs
```

## Architecture

UI lives in `src/features/<feature>/`, each with a fixed layout:

- `index.tsx` — the page. Composition + local state only. No `useEffect`, no fetching.
- `hooks.ts` — every hook with time-dependent behavior (`useEffect`, queries, …).
- `utils.ts` — pure helpers and static content. No React.
- `types.ts` — domain models and prop contracts. No inline `any`.
- `partials/` — presentational components: props in, callbacks out.

Routes are thin wrappers that mount a feature page. Cross-cutting code lives in
`src/lib/` (app-wide infrastructure) and `src/shared/` (shared domain); features
never import each other. The rules are checked by
[`scripts/validate-features.mjs`](./scripts/validate-features.mjs) via `npm run lint:arch`.

```
src/
├── router.tsx                 # getRouter() — TanStack Start entry
├── index.css                  # Panda CSS layer directives (reset, tokens, recipes…)
├── styled-system/             # generated by `panda codegen` (gitignored-friendly)
├── components/ui/             # primitives: Button, Card, Badge, Input, Heading,
│                              #   Text, Stack, HStack, Section (slot components)
├── shared/jobs.ts             # shared domain: Job/Application/Stage + seed data
├── lib/applications.tsx       # app-wide applications store (context)
├── features/
│   ├── landing/               # marketing page (nav, hero, pricing, waitlist…)
│   ├── dashboard/             # app shell: sidebar + milestone bar
│   ├── resume/                # resume reframer
│   ├── discover/              # scored role grid
│   └── tracker/               # application Kanban
│       ├── index.tsx          #   page (wiring)
│       ├── utils.ts           #   pure helpers
│       └── partials/          #   presentational components
└── routes/
    ├── __root.tsx             # html shell, fonts, meta
    ├── index.tsx              # → LandingPage
    ├── app.tsx                # → DashboardLayout + <Outlet/>
    ├── app.index.tsx          # /app → /app/resume redirect
    ├── app.resume.tsx         # → ResumePage
    ├── app.discover.tsx       # → DiscoverPage
    └── app.tracker.tsx        # → TrackerPage
```

`src/routeTree.gen.ts` is generated by the TanStack Router plugin on dev/build.
`styled-system/` is generated by `panda codegen` (or automatically by the PostCSS
plugin during `vite dev`/`vite build`).
